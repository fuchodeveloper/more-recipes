    More-recipes API Documentation

    Welcome to the More Recipes API! You can use our API documentation as a guide to access more-recipes API endpoints, which give you access to available recipes.

    More-Recipes provides a platform for users to share the awesome and exciting recipe ideas they have invented or learnt. Suppose a user comes up with a recipe, he/she can post it on More-Recipes and get feedback in form of reviews and votes from other users who explore that recipe. Users can also keep a list of their favorite recipes on the application.

    NB: Code examples are represented on the right hand side of this documentation. These provide sample response for the provided endpoints

    Authentication

    More recipes requires authentication to access the protected API routes. On successful sign up or sign in, a token is generated for the user to access protected routes.

    Sign up

    Request

    • Endpoint: POST /api/v1/users/signup
    • Body (application/json)

    Request:

    {
    "firstName": "john",
    "lastName": "doe",
    "emailAddress": "john@gmail.com",
    "password": "password",
    "confirmPassword": "password"
}

    Response

    • Status: 201:Created
    • Message: Signup successful
    • Body (application/json)

    Response:

    {
    "message": "Signup successful",
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJpZCI6NCwiZmlyc3ROYW1lIjoibWFyeSIsImxhc3ROYW1lIjoiZG9lIiwiZW1haWxBZGRyZXNzIjoibWFyeWRvZUBnbWFpbC5jb20iLCJpYXQiOjE1MTYxOTIzMjgsImV4cCI6MTUxNjI3ODcyOH0.O6OlhjvQUd_SIX_XEwtkdR2j-3vTR1js0iUFBE5rTpY"
}

    This endpoint is used to create a new user account.

    HTTP Request

    POST https://more-recipes-application.herokuapp.com/api/v1/users/signup

    Sign in

    Request

    • Endpoint: POST /api/v1/users/signin
    • Body (application/json)

    Request:

    {
    "emailAddress": "john@gmail.com",
    "password": "password"
}

    Response

    • Status: 200:OK
    • Message: Signin successful
    • Body (application/json)

    Response:

    {
    "message": "Log in successful",
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJpZCI6NCwiZmlyc3ROYW1lIjoibWFyeSIsImxhc3ROYW1lIjoiZG9lIiwiZW1haWxBZGRyZXNzIjoibWFyeWRvZUBnbWFpbC5jb20iLCJpYXQiOjE1MTYxOTIzMjgsImV4cCI6MTUxNjI3ODcyOH0.O6OlhjvQUd_SIX_XEwtkdR2j-3vTR1js0iUFBE5rTpY"

    HTTP Request

    POST https://more-recipes-application.herokuapp.com/api/v1/users/signin

    Recipes

    Get All recipes

    https://more-recipes-application.herokuapp.com/api/v1/recipes

    The above command returns JSON structured like this:

    {
    "recipes": [
        {
            "id": 1,
            "recipeImage": null,
            "recipeQuantity": 1,
            "recipeIngredient": "tomatoes",
            "recipeDirection": "Pour the chicken stock and the tomato stew into a sizeable pot and leave to boil."
        }
    ],
    "error": false
}

    This endpoint retrieves all available recipes properly paginated.

    HTTP Request

    GET https://more-recipes-application.herokuapp.com/api/v1/recipes

    Errors

    The more recipes API uses the following error codes:

    Error Code Meaning
    400 Bad Request -- You sent a bad request to our server.
    401 Unauthorized -- You are not authorized to perform action.
    403 Forbidden -- The action requested is hidden for administrators only.
    404 Not Found -- The specified recipe could not be found.
    405 Method Not Allowed -- You tried to access a recipe with an invalid method.
    406 Not Acceptable -- You requested a format that isn't json.
    410 Gone -- The recipe requested has been removed from our servers.
    429 Too Many Requests -- You're requesting too many recipes.
    500 Internal Server Error -- We had a problem with our server. Try again later.
    503 Service Unavailable -- We're temporarily offline for maintenance. Please try again later.
