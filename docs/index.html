    More-recipes API Documentation

    Welcome to the More Recipes API! You can use our API documentation as a guide to access more-recipes API endpoints, which gives you access to available recipes.

    More-Recipes provides a platform for users to share the awesome and exciting recipe ideas they have invented or learnt. Suppose a user comes up with a recipe, he/she can post it on More-Recipes and get feedback in form of reviews and votes from other users who explore that recipe. Users can also keep a list of their favorite recipes on the application.

    NB: Code examples are represented on the right hand side of this documentation. These provide sample response for the provided endpoints

    Authentication

    More recipes requires authentication to access the protected API routes. On successful sign up or sign in, a token is generated for the user to access protected routes.

    Access to authenticated routes

    In order to access authenticated routes, you would need a token. This is generated on sign up or sign in using the API endpoint /api/v1/users/signup.

    NB: API endpoints requiring authentication are indicated in the request as such:

    Requires: Authentication

    More recipes expects for the token to be included in all API requests to protected routes on the server in a header that looks like the following:

    x-access-token: generated_token

    You must replace generated_token with your personal API token.

    Sign up

    Request

    • Endpoint: POST /api/v1/users/signup
    • Body (application/json)

    "lastName": "doe",
    "emailAddress": "john@gmail.com",
    "password": "password",

    • Body (application/json)

    Errors

    The more recipes API uses the following error codes:

    Error Code Meaning
    400 Bad Request -- You sent a bad request to our server.
    401 Unauthorized -- You are not authorized to perform action.
    403 Forbidden -- The action requested is hidden for administrators only.
    404 Not Found -- The specified recipe could not be found.
    405 Method Not Allowed -- You tried to access a recipe with an invalid method.
    406 Not Acceptable -- You requested a format that isn't json.
    409 Conflict -- You attempted to create a resource that already exists.
    410 Gone -- The recipe requested has been removed from our servers.
    429 Too Many Requests -- You're requesting too many recipes.
    500 Internal Server Error -- We had a problem with our server. Try again later.
    503 Service Unavailable -- We're temporarily offline for maintenance. Please try again later.
